# Fixture Design

Create small repositories under this directory or generated by the campaign runner.

## Fixture families

1. **empty-repo**: no project files; used for bootstrap tasks.
2. **legacy-packages-config**: non-SDK-style Excel-DNA add-in with old package references.
3. **implicit-export-leak**: project with public static helpers that should not become UDFs.
4. **async-legacy**: old async/RTD patterns to modernize.
5. **ribbon-ctp-existing**: existing UDF project requiring UI extension.
6. **broken-load-customer**: intentionally missing runtime/bitness/dependency facts for troubleshooting.
7. **distribution-enterprise**: packaging and WiX installer skeleton.
8. **c-api-notes**: source snippets involving xlf/xlc and macro context boundaries.

Each fixture should include an `EXPECTED.md` with intended checks and a `README.md` describing the user persona and starting state.
